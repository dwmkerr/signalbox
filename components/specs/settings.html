Always applied to the jumplist and menu bar. #tag matches a tag, plain text matches the session name or its last message, ! excludes - e.g. #work !project shows work and hides anything named or tagged project. Blank shows all.

What this is

The app's settings window. A plain macOS preferences window: a popup for the icon choice, checkboxes for toggles, a text field for the filters, short grey captions under each control. Keep it small; new settings are new rows.

Additional filters

The filter field narrows the board everywhere - the jumplist and the menu bar dropdown share it, so the two never disagree. Tokens are space- or comma-separated: #tag matches a tag, plain text matches the session name or its last message, and a leading ! excludes either. A session shows when it matches every include token and no exclude token. The field applies live as you type (no Save button) and persists across restarts (tagFilter below). It is also settable at launch: open Signalbox.app --args --filter demo restarts straight into a filtered view, and --filter with no value clears it back to all.

The jumplist shortcut

Today the shortcut is shown as a caption: ⌃⌥J by default, overridden with defaults write com.dwmkerr.signalbox hotkey "cmd+shift+space". If the combo is taken by another app, signalbox falls back and notifies. macOS has a well-defined pattern for a proper control here: a recorder well, as in System Settings under Keyboard Shortcuts - click the well, it focuses, press a combination, it captures it (and shows a clear button). The de facto implementation is the KeyboardShortcuts Swift package, which provides the recorder control, persistence, and the global listener in one - it would replace our hand-rolled Carbon code and the defaults-write override.

Writes to the shared file merge key by key, so unknown keys survive. A missing or broken file means defaults, and must never break the hook path.

The /clear toggle

Claude Code fires SessionEnd with reason clear when you type /clear.

  • Checked (default): the session ends and leaves the board, the same as quitting.
  • Unchecked: the hook maps it to done instead, so the old session stays listed with its last exchange until you hide it (⌃X), remove it (⌃⌫), or it expires. The fresh post-clear session appears as a new row alongside it.

For scripting, SIGNALBOX_CLEAR_ENDS=0|1 overrides the file.
